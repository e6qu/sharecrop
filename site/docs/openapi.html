OpenAPI reference

Generated from the route registrations in internal/http/server.go by internal/openapi (go run ./cmd/sharecrop generate openapi, checked by make check-openapi). Method, path, operationId, and whether a route requires a bearer access token are derived from the actual route table and the local call graph, not hand-maintained. Request/response body schemas are derived from the actual Go DTO struct each handler decodes/writes; a route whose handler falls outside the standard decode/write patterns gets a generic placeholder instead of a guess (see the plain tag in the Schema column below). See the hand-written HTTP API reference for prose per-route request/response descriptions, or fetch the raw document at ./openapi.json.
